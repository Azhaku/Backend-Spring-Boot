Spring Boot: IPL Records Project

Project Type: BackEnd

Status: Attempted

Requirements

Implement REST APIs to list the names of the teams who won in the IPL cricket match and the teams who played in 2013.

Entity Details

The Teams entity should have the following attributes:

  • teamId: The unique ID of the team (Integer).
  • teamName: Name of the team (String).
  • runs: Runs made by the team (Integer).
  • targetRuns: Target runs they have to make (Integer).
  • year: Year in which the team played (Integer).

Here is an example of a Team JSON object:

[ { "teamId": 1, "teamName": "RR", "runs": 185, "targetRuns": 170, "year": 2011 }, { "teamId": 2, "teamName": "KKR", "runs": 125, "targetRuns": 190, "year": 2013 }, { "teamId": 3, "teamName": "RCB", "runs": 195, "targetRuns": 180, "year": 2013 } ]

API Routes

API Route API Type Success Response Code Validation Error Code
/teams/winning-teams GET 200 404
/teams/team-2013 GET 200 404

Tasks

Task 1: Service Layer Implementation in TeamsService

Implement the business logic in the service layer using TeamsRepository.

  • Method Details:
    • getAllTeams(): Implement the GET method, which should list all the teams' details.

Task 2: REST API Endpoints in TeamsController

GET request to /teams/winning-teams: Filter out the teams that have a number of runs greater than or equal to their target runs.

Response Body: JSON object representing the details of the winning team.

HTTP Status Code:

  • 200 (OK) - If winning teams found.
  • 404 (Not Found) - If no winning teams are found.

GET request to /teams/team-2013: Filter out the teams that belong to the year 2013.

Response Body: JSON object representing the details of the team.

HTTP Status Code:

  • 200 (OK) - If the teams for the year 2013 found.
  • 404 (Not Found) - If no teams are found for the year 2013.
